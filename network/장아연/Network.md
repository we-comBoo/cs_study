## 네트워크
* 2대 이상의 컴퓨터를 서로 연결되어 통신할 수 있도록 망이 구축된 것

## 인터넷
* 가장 규모가 큰 네트워크
* 인터넷을 통해 서로 다른 네트워크에 속해도 데이터 주고 받을 수 있음

## 프로토콜
* 서로 다른 하드웨어 (네트워크 통해 통신 가능한 기기) 간 데이터 통신 규약

## MAC 주소
* 네트워크 상 서로 구별을 위해 기기마다 고유하게 할당된 물리적 주소
* 컴퓨터에 장착된 LAN 카드 식별 번호


## IP 주소
* LAN 카드에 연결된 회선의 주소
* 고정되지 않음 - 인터넷망 접속 시마다 달라짐

## IP 주소 or MAC 주소로만 통신하면 안되는가?
* 실질적인 통신은 변경 불가능한 MAC 주소를 통하고 논리적인 IP 주소를 통해 최적의 경로 찾는 라우팅 진행 
* IP 주소로 목표된 네트워크로 패킷을 보낸 뒤 해당 IP에 등록된 MAC 주소로 변환되어 해당 기기로 패킷 전송
* IP 주소가 없는 경우, ISP 내 모든 router는 모든 서버의 MAC 주소를 가지고 있으며 라우팅 해줘야 함
    ```
    주민번호 : MAC 주소
    이름 : IP 주소
    별명 : 호스트 이름
    ```

## ARP 
* Address Resolution Protocol
* IP 주소와 MAC 주소를 매핑하는 절차
* ARP가 MAC 주소 알아오면 PC의 ARP 테이블에 저장

## 각 구성 장비 Table
* PC : ARP table
* Switch : Mac table
* Router : Routing table, ARP table 
## 네트워크 통신 방식
1. 유니캐스트
* 특정 대상과 1:1 통신
* 특정 목적지 주소로 전송

2. 멀티캐스트
* 다수와 1:N 통신
* 특정 그룹/다수에게 전송

3. 브로드캐스트
* 네트워크에 있는 모든 대상과 통신
* 불특정 다수에게 전부 전송

## 네트워크 종류 
### 규모 따른 분류
1. LAN - 근거리 통신망
* 한정된 좁은 지역에 구성된 네트워크
* 높은 초기 투자 비용
* 낮은 유지 보수비
* 구성장비 : switch, hub

2. WAN - 원거리 통신망
* 넓은 지역 연결하는 네트워크
* LAN과 LAN을 서로 연결하는 광역 네트워크
* ISP 업체가 관리 담당
* 구성장비 : router
